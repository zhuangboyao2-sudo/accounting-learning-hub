# 每年 1 月：新年度稅務參數更新

對應 [DEVELOPMENT_PLAN.md](../../DEVELOPMENT_PLAN.md) §8.1、§8.2、§6.3。目標：讓全站稅務數字在新年度開始前更新到位，並清空逾期複驗清單。

## Checklist

- [ ] 1. 以 WebSearch 查證財政部（etax.nat.gov.tw）與全國法規資料庫（law.moj.gov.tw）當年度最新公告的下列數字：
  - 綜合所得稅：免稅額、標準扣除額、薪資所得特別扣除額、課稅級距與稅率
  - 營利事業所得稅：稅率、免稅額起徵門檻、擴大書審適用標準
  - 營業稅：稅率、起徵點、零稅率與免稅範圍是否有異動
  - 各類所得扣繳率標準：各類所得扣繳率、免扣繳門檻（2,000 元門檻是否調整）
  - 二代健保補充保費：費率、起扣門檻
  - 勞保、就保、職災保險：費率、投保薪資分級表
  - 基本生活費、基本工資（影響最低工資相關扣繳與計算）
  - 各稅目申報期限：是否因該年連續假日順延（§6.2 精神：數字一律以官方公告為準，不得憑記憶填寫）
- [ ] 2. 建立新年度參數檔 `content/tax-parameters/<年度>.json`，比照 `content/tax-parameters/2026.json` 的欄位結構填入查證後數字，並在對應內容的 `sources` 標明官方來源網址
- [ ] 3. 執行 `npm run validate-content`，取得 `verified_at` 超過 12 個月的「待複驗清單」
- [ ] 4. 對清單中每一個檔案，重新走 §6.2 查證流程：
  - 法規未變 → 僅更新 `verified_at` 為當次查證日期
  - 法規有修正 → 修正內容數字，並於文末加註「修法紀錄」（原規定 → 現行規定 → 生效日）
- [ ] 5. 待複驗清單清空後，再次執行 `npm run validate-content` 確認通過
- [ ] 6. 同步更新速查工具區的「當年度申報期限總表」（`content/glossary.json` 或對應頁面資料來源）與 `public/templates/` 內含年度數字的 Excel 範本
- [ ] 7. 執行 `npm outdated` 檢視相依套件，小版本可直接升級；major 升級僅在安全性需要或阻礙開發時進行，且需另開分支（§8.5）
- [ ] 8. 全部完成後 commit，並在對應 GitHub Issue 留言記錄本次執行結果，關閉 Issue

## 完成標準

- [ ] 新年度 `tax-parameters/<年度>.json` 已建立且通過 validate-content
- [ ] 待複驗清單清空
- [ ] 申報期限總表與範本年度數字已同步
