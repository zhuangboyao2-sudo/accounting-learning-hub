# 每季（4／7／10 月）：數據驅動的內容自我優化

對應 [DEVELOPMENT_PLAN.md](../../DEVELOPMENT_PLAN.md) §8.3。1 月的年度循環已涵蓋當季優化，因此排程只在 4、7、10 月提醒，全年共 4 次。**此循環需 Phase 4 儀表板「學習分析報告」匯出功能上線後才能執行；若該功能尚未完成，本 checklist 暫不適用，於 Issue 留言註明並關閉即可。**

## Checklist

- [ ] 1. 由使用者於儀表板匯出本季「學習分析報告」（Markdown），內容含：
  - 各章節答題錯誤率排行
  - 重複答錯 ≥3 次的題目
  - SRS 中反覆遺忘（lapse 數高）的複習卡
  - 超過 60 天未讀的章節
  - 被標記「看不懂」的詳解
- [ ] 2. 將報告交給執行模型，依報告內容執行以下優化：
  - 錯誤率最高的 3 個章節：改寫教材（補充傳產實例、白話化說明）並增加練習題
  - 反覆遺忘的複習卡：拆成更小的記憶單位、重寫記憶點
  - 被標記「看不懂」的詳解：重寫詳解
  - 久未讀章節：判斷是內容問題（太難、太枯燥）還是排序問題（章節順序不合理），據以調整
- [ ] 3. 優化後的內容數字如涉及法規，同樣走 §6.2 查證流程
- [ ] 4. 執行 `npm run validate-content` 確認通過
- [ ] 5. commit，並在 GitHub Issue 留言記錄本次優化的章節與具體改動

## 完成標準

- [ ] 本季報告中錯誤率最高的章節已完成改寫
- [ ] 反覆遺忘卡片與「看不懂」詳解已處理
