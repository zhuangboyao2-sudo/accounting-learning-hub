# 每年 12 月：歷屆試題匯入與年度檢討

對應 [DEVELOPMENT_PLAN.md](../../DEVELOPMENT_PLAN.md) §8.1、§6.4、§8.4。記帳士考試通常於每年 11 月舉行，12 月可匯入當年度最新試題並執行年度檢討。

## Part A：歷屆試題匯入（§6.4）

- [ ] 1. 依 [past-exam-sources.md](past-exam-sources.md) 記錄的入口網址，至考選部考畢試題查詢平臺下載當年度記帳士 4 個專業科目的試題與公告答案 PDF
- [ ] 2. 執行 `scripts/import-past-exam.ts`（或當時的匯入輔助腳本）將 PDF 轉為初步 JSON 骨架
- [ ] 3. 選擇題全數收錄；申論／計算題收錄題幹與參考擬答方向，`type` 欄位標記為 `essay`（僅供閱讀不計分）
- [ ] 4. 逐題由 AI 補寫詳解，詳解中出現的法規數字比照 §6.2 走查證流程；若該題所依法規已修法，於詳解中加註「本題依當年法規，現行規定為…」
- [ ] 5. 每題填入 `material_ref` 連回對應教材節；標明出處 `source: {"type": "past-exam", "year": <年度>, "number": <題號>}`
- [ ] 6. 執行 `npm run validate-content` 確認新增題目通過 schema 驗證
- [ ] 7. 隨機抽查 10 題，核對本站收錄答案與考選部公告答案是否一致
- [ ] 8. commit

## Part B：年度檢討（§8.4）

- [ ] 1. 檢視儀表板「學習分析報告」全年度累積數據（若 Phase 4 功能已上線）：各章節錯誤率、反覆遺忘的複習卡、久未讀章節
- [ ] 2. 檢討本年度實際使用頻率低的功能，評估是否精簡
- [ ] 3. 檢視 [DEVELOPMENT_PLAN.md](../../DEVELOPMENT_PLAN.md) §7「未來擴充」清單，是否有項目具備啟動條件（例如使用者決定報考年度後，啟動讀書計畫排程）
- [ ] 4. 若有重大技術變更需求（換框架、加後端、引入付費服務），先在變更紀錄寫明理由與影響範圍，經使用者確認後才排入下年度工作
- [ ] 5. 將檢討結論寫入 [DEVELOPMENT_PLAN.md](../../DEVELOPMENT_PLAN.md) 文末變更紀錄

## 完成標準

- [ ] 當年度歷屆試題至少涵蓋最新一次考試的 4 個科目，且抽查 10 題與官方答案一致
- [ ] 年度檢討結論已記錄於變更紀錄
