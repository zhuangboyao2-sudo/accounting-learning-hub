## 勞退新制：雇主強制提繳6%

勞工退休金新制（個人退休金專戶制）規定，雇主應為適用勞動基準法之本國籍勞工，按月提繳不低於其**月提繳工資<TaxParam path="laborPension.employerContributionRate" format="percent" />**之退休金，存入勞工在勞保局設立的個人退休金專戶。這筆提繳金額**全額由雇主負擔**，不像勞保費、健保費有員工分攤的部分。

```
雇主每月提繳金額 = 月提繳工資 × 6%
```

員工可另外選擇自願提繳，提繳率為0%至6%不等，自願提繳部分從員工薪資中扣除、存入同一個人退休金專戶，且依所得稅法規定可自當年度個人綜合所得總額中全數扣除（不計入薪資所得課稅）。

## 「月提繳工資」與「投保薪資」是兩張不同的級距表

實務上常見的混淆點：勞保投保薪資、健保投保薪資、勞退月提繳工資，是**三張各自獨立的分級表**，金額級距並不完全相同：

```
勞保投保薪資級距：下限11,100元、上限45,800元
健保投保薪資級距：下限29,500元、上限313,000元
勞退月提繳工資級距：下限1,500元、上限150,000元
```

勞退月提繳工資級距的上限（15萬元）遠高於勞保與健保，這代表高薪員工的勞退提繳基礎，通常會比勞保、健保的投保薪資基礎來得高——記帳士或人資在設定員工投保資料時，須分別對照三張級距表，不能用同一個「投保薪資」數字套用在三種計算上。

## 計算範例

某員工月提繳工資32,000元（對應勞退月提繳工資分級表之級距），未自願提繳：

```
雇主每月提繳金額 = 32,000 × 6% = 1,920元
```

若該員工同時選擇自願提繳3%：

```
員工每月自願提繳金額 = 32,000 × 3% = 960元（從薪資中扣除）
個人退休金專戶每月合計入帳 = 1,920（雇主）＋ 960（員工自願）= 2,880元
```

## 提繳工資的認定：實際工資，非底薪

月提繳工資之認定應以勞工當月實際工資對照分級表，包含本薪、按月給付之固定津貼、獎金等經常性給與，而非只看勞動契約上約定的底薪數字——若員工實際月薪因加班費、績效獎金等經常性給與而提高，提繳工資也應隨之調整級距，不能長期固定按底薪最低級距提繳，否則將構成短提繳，影響員工退休金權益，也可能遭主管機關裁罰。

## 常見錯誤

- 誤以為勞保、健保、勞退可以用同一個「投保薪資」數字處理，忽略三者是各自獨立的分級表，級距上下限都不同
- 誤以為勞退提繳工資只看勞動契約約定的底薪，忽略應按員工實際經常性工資（含固定津貼、獎金等）對照分級表提繳
- 誤以為員工自願提繳部分要計入薪資所得課稅，忽略依規定可自個人綜合所得總額中全數扣除，不計入薪資所得
